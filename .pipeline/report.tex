%% ── 1. PROJECT OVERVIEW ────────────────────────────────────────────────────
\section{Project Overview}

Effective hyperparameter optimization is crucial for building robust machine learning models, particularly when executing complex regression tasks. This project addresses the need for efficient hyperparameter tuning in the context of two significant real-world tasks: improving the quality of 3D-printed parts and minimizing PLL loop filter lock time during post-silicon validation. By implementing Grid Search, Random Search, and Bayesian Optimization using the Optuna framework, we could refine machine learning models that exhibit both speed and accuracy in their predictions. 

Our approach involved exhaustive tuning of hyperparameters across both datasets, utilizing 5-dimensional search spaces for optimal parameter settings. For the 3D printer extrusion quality dataset, we aimed to maximize print quality score (ranging from 0 to 1) based on features like temperature, speed, and layer height. Concurrently, for the PLL loop filter tuning, our goal was to minimize the lock time expressed in microseconds (µs). The use of Optuna introduced advanced techniques such as surrogate modeling and intelligent sampling, which allowed for efficient exploration of complex parameter landscapes, resulting in improved model performance metrics.

The results demonstrated significant advancements in R² values, indicating excellent explanatory power of our models. Both methodologies were rigorously compared, providing insights into their effectiveness and efficiency in hyperparameter optimization, ultimately driving better model performance in the targeted tasks.

\subsection{Key Concepts}

\textbf{Grid Search:} An exhaustive technique that systematically evaluates every combination of hyperparameters in the defined search space.

\textbf{Random Search:} This method involves sampling a fixed number of random combinations of hyperparameters, allowing for quicker explorations of larger spaces compared to grid search.

\textbf{Bayesian Optimization:} A probabilistic model-based approach which builds a surrogate model to guide the search for optimal parameters, focusing on areas in the parameter space likely to produce better results.

\textbf{Optuna:} An automated hyperparameter optimization framework that applies advanced strategies like tree-structured Parzen estimators (TPE) for effective searching.

\textbf{Parameter Importance:} A technique used to identify which hyperparameters significantly impact model performance, providing insights into model interpretability.

\textbf{Surrogate Models:} Statistical models that approximate the true objective function used in Bayesian optimization to predict outcomes based on hyperparameter inputs.

\textbf{Acquisition Functions:} Functions that decide where to sample in the hyperparameter space based on prior evaluations, optimizing the balance of exploration and exploitation.

\subsection{Processing Pipeline}

The processing pipeline began with data loading from CSV files for both datasets, ensuring correct format handling via pandas. Exploratory Data Analysis (EDA) was conducted to visualize data distributions, identify correlations among features, and detect any outliers that might affect model performance. Following this, feature engineering steps included normalization and scaling of input features to ensure optimal performance of regression algorithms.

Then, baseline models were trained using linear regression to establish initial performance metrics and form a foundation for comparative analysis. Moving forward, hyperparameter optimization was implemented using the aforementioned techniques, where Grid Search provided an exhaustive evaluation for smaller search spaces, while Random Search allowed for quicker sampling within larger domains. Bayesian Optimization with Optuna was employed, leveraging advanced sampling techniques to explore hyperparameter interactions comprehensively.

Finally, each model's performance was evaluated using metrics such as R² and RMSE, and results were analyzed to determine which hyperparameters contributed significantly to improvements. Visualizations were created to illustrate optimization paths and performance outcomes, providing deeper insights into the models' behaviors.

%% ── 4. RESULTS ─────────────────────────────────────────────────────────────
\section{Results}

The experiments yielded an impressive R² value of 0.9741 for the grid search variant on the 3D printer dataset, indicating exceptional predictive capability. Random Search also performed admirably, achieving an R² of 0.9733, suggesting that, despite fewer evaluations, it effectively explored meaningful combinations of parameters. Bayesian Optimization with Optuna proved its worth particularly on the PLL loop filter tuning dataset, where a minimum lock time was recorded after employing a TPE sampler, showcasing its efficiency in navigating larger hyperparameter spaces. 

A comparative analysis revealed that, while Grid Search provides exhaustive results, its computational cost scales unfavorably with increasing parameter dimensions. Meanwhile, Random Search exhibited advantages in computation time at the expense of thoroughness, leading to mixed performance. Optuna outperformed both in terms of efficiency and end results, particularly in the larger and more complex PLL tuning task, emphasizing the importance of choosing the right optimization approach based on specific problem characteristics.
